soap service vs rest service

what do you mean by api ?

api is a way of communication between

frontend and the backend(database) of the application

two different/same technologies applications

two different platform applications

we have two type of services(APIs)

soap Service
rest service

when development team has design the soap service for communication they will give us a single file named
wsdl(webservices description(definitation) language

all the operations/features/resources will be defined in the single wsdl file

we have to test that wsdl file


in case of soap service developer will give us wsdl file 
wsdl file will contain all the resources/features/operations


wsdl file is based on xml format


html format-----------------predefined tag
html head title body div

xml format----------------user defined tag
<firstname>  </firstname>


in case of rest service 

there will be a unique URI(URL) for each operation/resource/functionality


sum-------------www.calculator.com/sum
sub---------------www.calculator.com/sub

rest service support both the formats
xml format and json format



xml format----------- <firstname>deepak</firstname>
json format---------"firstname" : "deepak"

URI----Uniform resource identifier----------

URI consist --------------Base + end point + parameter


http://localhost:3000/APIStudentsList

Base-----localhost:3000
end point--------/APIStudentsList


https://dummy.restapiexample.com/api/v1/employees

Base-----https://dummy.restapiexample.com/api/v1
end point-----/employees



before end point whatever is written that is the base location





https://dummy.restapiexample.com/api/v1/create

Base---https://dummy.restapiexample.com/api/v1
end point----/create


https://dummy.restapiexample.com/api/v1/employee/1

Base----
https://dummy.restapiexample.com/api/v1

end point------/employee

path parameter-----/1



parameter------------? or / 

? query parameter

/ path parameter


https://www.makemytrip.com/hotels/

Base----https://www.makemytrip.com

end point------/hotels



https://www.makemytrip.com/railways/
Base----https://www.makemytrip.com

end point------------/railways



& for multiple query parameter


resource/feature/operation/end point--------------functionality


soap service-----------wsdl file
rest service-------------URI
